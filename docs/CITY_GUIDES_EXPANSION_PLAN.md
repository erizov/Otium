# План: гиды по городам по модели Smolensk

Цель: для каждого города — отдельная папка, та же **логика данных и сборки**, что у `smolensk/`: реестр мест, JSON объектов + деталей, whitelist источников, плоский каталог `images/`, скрипты загрузки и `build_*_pdf.py` (титул с гербами/флагами и полосой «ведущих» учреждений при необходимости).

## Правила контента (как вы задали)

| Правило | Реализация |
|--------|------------|
| Нет текста по полю | Поле не заполнять или не включать в merge (как сейчас в `places_registry.py`: пустое не перетирает). |
| Нет **локального** изображения (файл в `images/` ≥ минимального размера) | Объект **не попадает в PDF** (и при желании не в HTML), но slug может остаться в JSON как кандидат. |
| Объектов в гиде | **от 20 до 150** — по факту доступных **проверенных** фото и текстов. |

Рекомендуемый целевой коридор по городам ниже — ориентир; итог = сколько мест прошло порог изображения после `download_*` и валидации.

## Эталонная структура папки города (как `smolensk/`)

```text
<city>/
  __init__.py
  whitelist.py
  image_optimize.py          # опционально: общий модуль или symlink на smolensk
  docs/
    SOURCES_WHITELIST.md
    CANDIDATES.md            # черновой список slug → статус фото
  data/
    <city>_places.json       # slug, category, name_*, image_*, optional additional_images
    <city>_place_details.json
    places_registry.py       # загрузка + merge details по slug
  images/                    # только плоские пути images/<slug>.jpg
  output/
    <city>_guide.html
    <city>_guide.pdf
```

Скрипты в `scripts/` (по аналогии):

- `download_<city>_images.py` — очередь из JSON + титульные ассеты.
- `validate_<city>_sources.py` — whitelist для `image_source_url`.
- `build_<city>_pdf.py` — HTML+PDF, фильтр «только места с локальной картинкой».

Титул: герб/флаг города или региона + опционально «витрина» университетов/институтов — **только если** есть файлы в `images/` и разрешённые URL (как у Смоленска/СПб).

## Общий пайплайн (один город)

1. **Выбрать 30–200 кандидатов** (категории Commons, списки ЮНЕСКО, официальные туристические списки).
2. **Зафиксировать slug** (латиница, snake_case) и **один** `image_source_url` на объект (желательно Commons `upload.wikimedia.org`).
3. **Расширить** `docs/SOURCES_WHITELIST.md` города при необходимости (музейные CDN, city administration — осторожно с лицензиями).
4. **Заполнить** `<city>_places.json` (минимум: slug, category, name_ru или name_en, `image_rel_path`, `image_source_url`).
5. **Заполнить** `<city>_place_details.json` по мере наличия фактов; пустые блоки не обязательны.
6. **`python scripts/download_<city>_images.py`** → затем опционально оптимизация изображений (порог размера).
7. **`python scripts/validate_<city>_sources.py`**
8. **`python scripts/build_<city>_pdf.py`** — в выборку попадают только места с успешно лежащими файлами.
9. **Проверка числа**: если &lt; 20 — доработать кандидатов; если &gt; 150 — отсечь по приоритету (иконичность, покрытие категорий).

## Категории (как у Smolensk / SPB)

Держать смесь: `landmarks`, `museums`, `parks`, `bridges`, `squares`, `theaters`, `places_of_worship`, `railway_stations`, `viewpoints`, `sculptures`, `markets`, `libraries`, `palaces`, `cemeteries`, `misc` — не все обязательны в каждом городе.

## Планы по городам

Имена папок — **латиница, нижний регистр**; ниже ориентир по целевому числу карточек **с картинкой**.

### 1. Paris (`paris/`) — цель 80–130

- **Контент:** исключительно плотный Commons; Лувр, Нотр-Дам, Эйфелева башня, Орсе, Сакре-Кёр, Триумфальная арка, Марсово поле, Латинский квартал, мосты через Сену.
- **Символика титула:** герб Парижа, флаг Парижа (если есть свободные файлы); опционально 2–3 университета (Сорбонна и др.) — только при стабильных Commons-файлах.
- **Риски:** популярные объекты — дубли несвободных фото; держаться **собственных работ** на Commons.

### 2. Rome (`rome/`) — цель 90–150

- **Контент:** Колизей, форум, Пантеон, Ватикан (снаружи/Собор Св. Петра — внимание к правилам съёмки), фонтаны, виды с холмов, мосты Тибра.
- **Символика:** герб Рима; флаг — проверить лицензию на выбранном файле.
- **Риски:** Ватикан и музеи — не все интерьеры свободны; приоритет **фасады и улицы**.

### 3. Barcelona (`barcelona/`) — цель 70–120

- **Контент:** Саграда Фамилия, Готический квартал, парк Гуэль, Дом Бальо, Мила, Монжуик, пляжная зона (1–2 вида), Камп Ноу (если есть свободные снимки фасада).
- **Символика:** герб/флаг Барселоны (в Commons есть варианты — проверить лицензии).

### 4. Madrid (`madrid/`) — цель 70–120

- **Контент:** Прадо (фасад), Королевский дворец, Пласа-Майор, Пуэрта-дель-Соль, парк Ретиро, станции и вокзал Аточа (архитектура), арена Лас-Вентас (снаружи).
- **Символика:** герб Мадрида, флаг автономного сообщества.

### 5. Lisbon (`lisbon/`) — цель 50–100

- **Контент:** Алфама, трамваи, мост 25 апреля (вид с города), Белен, башня Белен, монастырь иеронимитов, площади центра.
- **Символика:** герб Лиссабона; флаг города.

### 6. Venice (`venice/`) — цель 60–110

- **Контент:** Сан-Марко, Дворец дожей, мост Вздохов (снаружи), Риальто, Гранд-канал (типовые ракурсы), острова Мурано/Бурано (по 1–2 объекта).
- **Риски:** туристические фото с неверной лицензией — строгий whitelist и ручной отбор файлов.

### 7. Florence (`florence/`) — цель 60–110

- **Контент:** Дуомо, Понте Веккьо, Уффици (фасад), Площадь Синьории, Санта-Кроче, виды с Микельанджело.
- **Символика:** герб Флоренции.

### 8. Prague (`prague/`) — цель 70–120

- **Контент:** Карлов мост, Градчаны, Староместская площадь, астрономические часы, Влтава, Национальный театр, Вышеград.
- **Символика:** герб Праги, флаг города.

### 9. Budapest (`budapest/`) — цель 70–120

- **Контент:** Парламент, цепной мост, купальни Сечени, Будайская крепость, базилика Св. Иштвана, набережные.
- **Символика:** герб Будапешта.

### 10. New York City (`new_york/`, ориентир «NYC») — цель 80–130

- **Контент:** Статуя Свободы (общие ракурсы с Commons), Бруклинский мост, Таймс-сквер, Центральный парк, Эмпайр-стейт/Топ оф зе Рок (только если есть свободные снимки снаружи), музеи (фасады MET, MoMA).
- **Риски:** США — не везде «freedom of panorama» для современных зданий; **предпочитать Commons** с явной лицензией.

### 11. Boston (`boston/`) — цель 40–80

- **Контент:** Тропа свободы, Фанейл-холл, Кеймбридж/Гарвард (кампус с улицы), старый штат-хаус, набережная.
- **Символика:** флаг/печать города при наличии на Commons.

### 12. Los Angeles (`los_angeles/`) — цель 50–90

- **Контент:** Гриффит (обсерватория и вид), Голливудский знак (общие виды), пляж Санта-Моника, центр (небоскрёбы — проверка FoP), Getty Center (архитектура снаружи).
- **Риски:** частные объекты и торговые знаки — уклон в **панорамы и общественные места**.

### 13. Chicago (`chicago/`) — цель 50–90

- **Контент:** Миллениум-парк (Cloud Gate — проверить политику снимков на Commons), набережная, Арт-институт (фасад), Willis Tower (внешний вид), театр Chicago.
- **Риски:** те же, что NYC.

### 14. Montreal (`montreal/`) — цель 50–90

- **Контент:** Старый Монреаль, Нотр-Дам де Монреаль, Мон-Рояль, стадион Олимпийский (снаружи), музей изящных искусств.
- **Символика:** герб Монреаля; язык карточек — EN/FR по выбору редакции.

### 15. Berlin (`berlin/`) — цель 80–130

- **Контент:** Бранденбургские ворота, Рейхстаг (снаружи), остаток Берлинской стены (Мюленштрассе и др.), музейный остров (фасады), Александрплац, мемориалы (с уважением к тону гида).
- **Символика:** герб/символ Берлина (актуальный герб округа).

### 16. Jerusalem (`jerusalem/`) — цель 40–90

- **Контент:** Старый город (общие вида, ворота), Стена плача (с осторожностью к правилам съёмки и такту), Храмовая гора (дистанционные ракурсы, если допустимо), церкви и мечети **с улицы**.
- **Риски:** политическая чувствительность и религиозные ограничения; **нейтральные подписи**, только проверенные свободные изображения.

## Техническая унификация (следующий шаг после плана)

- Вынести **общий каркас** в `scripts/city_guide_template.py` или конфиг `cities.yaml` (корень, имена файлов, MIN bytes), чтобы не копировать 16 раз один и тот же код.
- Оставить **per-city** только: whitelist, JSON, title assets, CSS-тонкости титула.
- Единое правило: **`build_*_pdf` фильтрует места** через функцию «есть локальный файл по `image_rel_path` и optional same-stem» — частично: `scripts/city_guide_core.py` (`smallest_same_stem_image_rel`, пороги для SVG/GIF).

## Итог

Для каждого из 16 городов папка зеркалит `smolensk/`, объём **20–150** объектов определяется **после** загрузки изображений и сборки; отсутствующие поля текста допускаются; **без файла в `images/` место в PDF не входит**.
